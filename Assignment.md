1.What are App Services and what are the different types of options available in App Services.
   Microsoft App Services is a cloud service offered on Azure, which can be utilized for the development, deployment, and hosting of applications, while eliminating the need for managing any underlying server infrastructure.
   Various alternatives available include Web Apps, API Apps and  Mobile Apps.

2.Explain Cloud Availability Set and Availability Zones
  Availability Set spreads out virtual machines in various fault and update domains, thus minimizing the time lost due to maintenance operations or system failures.
  On the other hand, an Availability Zone refers to a physically distinct set of data centers within the same region.

3.What are the responsibilities of users when it comes to different cloud offerings like Infrastructure as Service, Platform as Service and Function as Service and mention which Azure resource falls under each option.
  In the case of IaaS, users have to take care of the operating system, applications, and the data, whereas in PaaS, users only need to manage the application and data. 
  In IaaS, the service provider takes care of the hardware while, in the case of PaaS, the service provider looks after the infrastructure itself.
  Finally, in FaaS, users only need to manage the code deployment, and other activities are managed by the cloud provider, like in Azure Virtual Machines in IaaS, Azure App Services in PaaS, and Azure Functions in FaaS.

4.In a brief paragraph, explain networking options available in Azure mentioning each service discussed in the class.
  There are many networking services offered by Microsoft Azure for securing and connecting cloud resources together.
  Virtual Network (VNET), which facilitates private communications between resources.
  Load Balancer, which directs traffic among numerous servers. 
  Virtual Private Network Gateway, which establishes secure connectivity from on-premise network to Azure networks. 
  Application Gateway, which manages and routes web traffic efficiently and safely and DNS in Azure for facilitating domain name resolution.

 5.What are storage accounts and the different options available to access storage accounts
   Azure storage account is one of the services offered by Microsoft Azure, which can be used to store various kinds of data including blobs, files, queues, and tables in the cloud environment. 
   There are different ways that users can access storage accounts, including through the Azure portal, Azure Storage Explorer, PowerShell and SAS.

 6.What are different options available to scale up and scale out Azure App Services
   App Services in Microsoft Azure can be scaled either by using Scale Up or Scale Out methods.
   Scale Up is related to enhancing the existing server's capacity (CPU, RAM, pricing tier, etc.) whereas Scale Out involves deploying more copies of the application. 
   This way, the performance of the application can be sustained.
